# newtua-formats

> Декодеры на чистом Rust для устаревших форматов архивов, о которых забыла современная экосистема.

Где-то на старом жёстком диске лежит `.sit` с Mac 1996 года, образ дискеты `.dms` с Amiga, `.arj` со старого DOS-BBS или `.zoo` из архива Usenet. Программ, которые их создали, давно нет, а нынешние Rust-крейты их не читают.

`newtua-formats` их читает. Это группа небольших самодостаточных крейтов, которые декодируют **именно те форматы, которые не нашлись в экосистеме Rust**. Алгоритмы распаковки аккуратно портированы из проверенного временем движка **XADMaster** ([The Unarchiver](https://theunarchiver.com/)) и сверены с ним на корректность.

Проект создан как архивный «движок» для более крупной работы — **New The Unarchiver**, — но каждый крейт самодостаточен и предлагается широкому сообществу Rust и всем, кто занимается цифровой сохранностью.

## Зачем это нужно

- **В экосистеме вскрылись пробелы.** Массовые форматы в Rust покрыты хорошо (zip, 7z, tar, RAR, gzip, bzip2, xz, zstd, LHA через `delharc`, `.Z`). А вот *старьё* — архиваторы эпохи классического Mac, Amiga и DOS/CP-M — в основном, нет. Проект закрывает эти пробелы.
- **Старые данные заслуживают того, чтобы их можно было прочитать.** Эти форматы важны для ретрокомпьютинга, работы музеев и архивов и для всех, кто вытаскивает файлы с устаревших носителей.
- **Никакого FFI и внешних вызовов.** Всё на чистом Rust. Вам не нужны ни `unar`, ни `7z`, ни какая-либо C-библиотека во время работы.

## Как всё началось

Мы хотели дать [The Unarchiver](https://theunarchiver.com/) развитие — подарить этой заслуженной Mac-утилите, которая была заброшена её новыми хозяевами, современную **кроссплатформенную жизнь на Rust**. Пересобирая её, мы всё время упирались в одну и ту же стену: для массовых форматов хорошие Rust-крейты были, а для *легаси* — тех самых старых архиваторов Mac, Amiga и DOS, которыми и славился The Unarchiver, — не было ничего.

Тогда мы пошли к первоисточнику. Движок The Unarchiver, **XADMaster**, написан на Objective-C, и его код был открыт. (Позже The Unarchiver продали на сторону, но тот Objective-C-код к тому моменту уже давно был доступен под LGPL.) Вместо того чтобы разбирать каждый формат с нуля, мы портировали декодеры из этой эталонной реализации: читали проверенный ObjC, переписывали его как идиоматичный Rust без зависимостей и сверяли результат с оригиналом.

`newtua-formats` — это и есть итог: слой легаси-форматов, вынесенный в отдельные крейты, чтобы им могло пользоваться всё сообщество, а не только наш распаковщик.

## Часть проекта New The Unarchiver

Эти крейты — один из слоёв **[New The Unarchiver](https://github.com/new-the-unarchiver)**, кроссплатформенного перерождения The Unarchiver, написанного на Rust с нуля. Если оригинал был приложением для macOS, то New The Unarchiver задуман работать везде и добавляет кроссплатформенную командную строку, встроенный терминальный интерфейс (TUI), полностью внутрипроцессную распаковку (без внешних `unrar`/`7z`/`tar`), выборочное извлечение по маске (glob) и локализуемые сообщения — при этом читая те же форматы архивов, ради которых пользователи и держали оригинал. `newtua-formats` — это та часть движка, что отвечает за *легаси*-форматы. Мы вынесли её отдельно и опубликовали саму по себе, потому что хороший декодер забытого формата на чистом Rust полезен далеко за пределами одного приложения — и вернуть его сообществу и есть смысл затеи.

|--------|-------|-------|--------------------------|

## Что мы не стали портировать — и почему

Кое-что отсутствует намеренно — чтобы границы были ясны сразу.

**Массовые форматы вне области проекта** — Rust с ними и так хорошо справляется, и дублировать эту работу нам незачем: zip, 7z, tar, RAR, gzip, bzip2, xz, zstd, LHA/LZH (через `delharc`), `.Z`. `newtua-formats` закрывает пробелы, а не переизобретает то, что уже есть.

**Несколько легаси-форматов отложены** — сегодня они по-настоящему редкие, вернёмся к ним по мере спроса:

| Формат | Почему пока нет |
|--------|-----------------|
| DiskDoubler (`.dd`) | Классический Mac начала 90-х, только глубокое ретро. Много методов (часть не проверена даже в первоисточнике) плюс сжатие Stac LZS |
| NowCompress | Ещё более нишевый классический Mac; опирается на хрупкие эвристики смещений, для проверки которых сперва нужен реальный корпус |
| Прочие редкие форматы libxad | Сегодня встречаются крайне редко |

**О шифровании и краевых случаях:** там, где исходный XADMaster / `unar` не умеет расшифровать или извлечь данные, не умеем и мы — возвращаем честную ошибку `Unsupported`, а не пытаемся угадать. Приятные исключения — **ALZip** и **DMS**, где мы идём *дальше* эталона и всё-таки расшифровываем/извлекаем, сверяясь с независимыми инструментами.

## Использование

Подключите крейт нужного вам семейства:

```sh
cargo add newtua-dos
```

Почти у всех форматов один и тот же небольшой единообразный API — `recognize`, чтобы распознать; `open`, чтобы разобрать; `entries`, чтобы перечислить; `read_entry`, чтобы извлечь:

```rust
use std::io::Cursor;
use newtua_dos::zoo::ZooArchive;

fn main() -> std::io::Result<()> {
    let bytes = std::fs::read("classic.zoo")?;

    // Опционально: дёшево убедиться в формате, прежде чем запускать парсер.
    if !ZooArchive::recognize(&bytes) {
        eprintln!("это не архив Zoo");
        return Ok(());
    }

    let archive = ZooArchive::open(Cursor::new(bytes))?;
    for (i, entry) in archive.entries().iter().enumerate() {
        if entry.is_dir() {
            continue;
        }
        let name = String::from_utf8_lossy(entry.name());
        let mut out = Vec::new();
        archive.read_entry(i, &mut out)?;
        println!("{name} — {} байт", out.len());
    }
    Ok(())
}
```

Точно так же работают `newtua_mac::binhex::BinHexArchive`, `newtua_stuffit::stuffit::StuffItArchive`, `newtua_alz::AlzArchive`, `newtua_nsis::NsisArchive` и остальные.

**Имена файлов — это байты.** `entry.name()` возвращает `&[u8]`, а не `String`: эти архивы старше UTF-8 и хранят имена в Mac Roman, Shift-JIS, Amiga Latin-1 и других кодировках. Декодирование кодировки оставлено вызывающей стороне, которая знает происхождение файла, — сознательный выбор ради честной сохранности.

**Несколько потоковых форматов устроены иначе.** Тройка Amiga построена вокруг того, что реально хранит: `PowerPacker` декодирует один поток (`open(..).decode()`), а `DMS` — это образ дискеты, который читают целиком (`read_disk_image()`) или по файлам (`read_file(..)`). Подробности — в документации каждого крейта на docs.rs.

## Корректность

У большинства этих форматов **не сохранилось ни одного компрессора**, поэтому просто «прогнать туда-обратно» через эталонный инструмент нельзя. Вместо этого каждый декодер проверяется тремя способами:

1. **Зеркальный кодировщик** — точная инверсия декодера, которой синтезируются тестовые данные для постоянного round-trip-теста.
2. **Оракул `unar`** — вывод побайтово сверяется с эталонным распаковщиком The Unarchiver.
3. **Независимые кодировщики** — там, где они есть (например, `binhex`, `macbinary`, Info-ZIP `zip`), данные готовятся и сторонними инструментами.

Весь проект написан через тесты, поэтому каждый формат поставляется вместе с тестами, которые его покрывают.

## Требования

- **Rust 1.74+** (edition 2021).
- Никаких системных зависимостей. Пара крейтов опирается на зрелые кодеки на чистом Rust, которые сознательно не переписываются (`bzip2-rs` для ALZip, `lzma-rs` для NSIS); оба под разрешительными лицензиями и полностью совместимы с LGPL.

## Происхождение и лицензия

**Под лицензией LGPL-3.0-or-later** — и, по правде говоря, это не был свободный выбор; это единственный честный вариант.

Алгоритмы распаковки портированы из **XADMaster** (The Unarchiver) авторства Dag Ågren и участников. Даг с самого начала строил весь движок под **LGPL**, а часть кода восходит к ещё более старому LGPL-коду, который перелицензировать нельзя никогда. Добросовестный порт такой работы — это производная работа, поэтому он наследует лицензию: законного способа перевести его под MIT или Apache не существует. Мы с этим не боремся — мы это принимаем. К тому же именно LGPL сохраняет эти декодеры *свободным ПО навсегда*, а это ровно дух оригинала.

- Полный текст: [`LICENSE`](LICENSE) (GNU LGPL v3), который включает по ссылке GNU GPL v3 из файла [`GPL-3.0.txt`](GPL-3.0.txt).
- Копирайт и происхождение: [`NOTICE`](NOTICE).

Проще говоря: библиотека остаётся свободным ПО навсегда, но — благодаря *Lesser* GPL — вы всё равно можете использовать её внутри программы под лицензией на ваш выбор.

## Статус

Перечисленные форматы реализованы и покрыты тестами. Несколько более редких вариантов остаются в очереди (см. *Что мы не стали портировать*); порядок реализации ведётся в сопутствующем проекте New The Unarchiver. Вклад и сообщения об ошибках по поддерживаемым форматам приветствуются.
